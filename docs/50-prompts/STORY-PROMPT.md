====================
【阶段划分规则（强约束）】
=============

必须基于以下信号划分阶段：

- 提交密度突变（爆发 / 收敛）
- commit 类型变化
- 关注点变化（功能 → 失败路径 → 语义 → 门禁 → 治理）

❌ 禁止：

- 固定阶段数量

====================
【每个阶段结构（必须包含）】
==============

1. 【触发点】
    - 描述“哪里开始不一样了”
    - 必须基于 commit 特征（密度 / 类型 / 主题）

2. 【关键动作】
    - 至少 2–3 类具体行为
    - 必须具体（如：幂等、死信、DTO 下沉、门禁规则）

3. 【冲突 / 反复（关键加分项）】
    - 如果存在：
        - 迁移 → 回退 → 重做
        - 规则 → 撤回 → 再收紧

    - 必须明确写出“这个阶段不稳定”

4. 【隐含判断（观察语气）】
    - ✅ “从这些提交看，更像是在……”
    - ✅ “这一变化通常意味着……”
    - ❌ 禁止写成“作者的意图是……”

---

====================
【故事张力要求（重点）】
============

必须至少识别并放大 2–3 个“关键波动点”，例如：

- Facade 来回迁移

- 不确定性

- 收敛过程
